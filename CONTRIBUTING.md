# 기여 가이드

## 이슈 작성

- 작업 전에 이슈를 만들고 목적, 배경, 완료 조건을 적어 주세요.
- 버그는 재현 절차, 기대 동작, 실제 동작, 실행 환경을 포함해 주세요.
- 기능 요청은 사용자 가치, 필요한 API/화면 영향, 완료 기준을 함께 적어 주세요.

## 브랜치 규칙

- `feature/이슈번호-작업명`: 새 기능
- `fix/이슈번호-작업명`: 버그 수정
- `docs/이슈번호-작업명`: 문서 변경
- `chore/이슈번호-작업명`: 설정, 의존성, 빌드 작업
- `refactor/이슈번호-작업명`: 동작 변경 없는 구조 개선

## PR 규칙

- PR 제목은 변경 내용을 짧고 명확하게 작성합니다.
- 관련 이슈를 `Closes #번호` 형식으로 연결합니다.
- 테스트 명령과 결과를 PR 본문에 남깁니다.
- 리뷰어가 집중해서 볼 부분이 있으면 별도로 표시합니다.

## 머지 기준

- CI가 통과해야 합니다.
- 최소 1명 이상의 승인을 받은 뒤 머지합니다.
- 머지 후 작업 브랜치는 삭제합니다.
